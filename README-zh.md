# collab-master-skill

collab-master 自己不动手。不写答案，不画图，不做 PPT。它只做一件事：在 AI 回答之前帮你把问题先看过一遍——要不要追问、是不是在编、换不换个角度、是不是走错了。

**一句话定位**：collab-master 是纵向调度器，管的是整个任务的形状和流程。superpowers 这类 skill 是横向纪律库，管的是某一个具体动作怎么做对。两个层，不重叠，互相补充——[下面有详细说明](#和其他-skill-的关系)。

当前版本 v1。v1.5 多角色辩论、v1.6 下游工具对接已经落地。

## 它干什么

你问 AI 一个问题。普通的 AI 张嘴就答。有时候答得很好，有时候跑了题，有时候缺信息就自己编一个补上，有时候给了个模板答案，看着挑不出毛病但换个问题答案也不变。

collab-master 插在中间。先看几件事：

- 你给的背景够不够？不够就追一个问题，带默认选项。没回就按默认往下走。
- 这问题有没有坑？要不要联网验证一下？标上标记，后面的步骤绕不过去。
- 你是谁？要干活还是想学东西？调深度和语气。
- 简单问题直接给结论。复杂问题先多想几个方向，再收束。
- 答案写完了，有没有编数字？有没有悄悄换了你的目标？三道闸，过了才发。


## 举个例子

你问："这段代码线上偶尔 500，帮我看看。"

装了 collab-master 的 AI 会先判断你要的是排查问题，不是让你解释 500 状态码。紧跟着问："有报错日志吗？"这个不问没法往下查。拿到日志之后，从至少 6 个方向列假设——输入边界、最近变更、环境差异、数据状态、并发、依赖链。按可验证性排好序，再给排查方案。

没装的 AI 大概率说"检查日志、加 try-catch、重启服务"。这三句话换个 bug 也能用，换个人问答案也一样。

## 它拦了什么

AI 回答问题时有一些固定动作。collab-master 对这些动作各设了一道检查。

**1. 没弄清你要什么就答**

意图解析先判断你的目标。你是想学原理，还是想让它帮你干活，还是想比较几个方案。不照字面直接回。

**2. 信息不够就自己编**

信息缺口分了三级。A 级，缺了这个没法做，停住问你。B 级，缺了影响质量但不歪大方向，标 `[假设:...]`。C 级，可有可无，跳过去。

**3. 把不确定的事写成确定的口吻**

所有没验证的说法都要标 `[待验证:...]`，别装成事实。

**4. 对谁都一个语气**

受众适配。同一个问题，领导收到的版本结论摆在最前面，学生收到的版本带推导过程，工程师收到的版本有技术细节。不是同一篇问答换个抬头。

**5. 只给第一个想到的答案**

先想至少 8 个不同的方向，挑其中最好的 3 个。至少有一个得是反直觉的，不能全都是顺着惯性想出来的。

**6. 编数字、编引用、编版本号**

三层质量闸。第一层，致命红线，编造数据直接打回去。第二层，这句话换个完全不同的项目还能直接用吗？删掉一句，剩下的还有信息量吗？有新东西吗？第三层，读起来像人说话还是像客服在念稿。

**7. 卡住了反复试同一招**

同一个策略两次不行就切。连续三轮卡在同一个地方，不继续死磕，交当前最好的版本，标出哪没做完。

**8. 上下文一长就忘了你最开始的要求**

约束不衰减。你的要求不是靠 AI 的记性，是存成结构化规则。每次回答前机器扫一遍，命中关键词就退回去重写。

**9. 任务越跑越长收不住**

token 快用完的时候，立即把当前最好的版本输出，标 `[未完成]`，不强撑。

**10. 关了对话就忘了你的习惯**

异步记忆。同一类任务被你纠正三次以上，记住这个偏好，下次自动用上。一次性的临时指示不记。

## 怎么工作

不是固定的流水线。更像一个按需调度的总控。简单问题只激活两三个模块就跑完了，复杂问题才把所有模块走一遍。

你的问题进来后：

1. 意图解析 (intake)。你到底要干嘛？太模糊或缺信息就先停下来问你。
2. 边界判断 (boundary)。有风险吗？要验证吗？信息够吗？同时定复杂度。低（查个事实、闲聊）跳过策略层和发散层。中（解释概念、给建议）加一层受众适配。高（做设计、排查问题）走完整流程。
3. 受众策略 (strategy)。你是谁，回答该长什么样。7 种模式。
4. 发散收敛 (solution-space)。多给方案还是直接结论？仅高复杂度触发。
5. 生成答案 (compose)。把前面各模块的判断写成你看到的回答。如果要出 PPT、图表、文档，从注册表找能做这件事的下游工具，材料转成对应格式扔过去。
6. 质量检查 (quality-gate)。三道闸。
7. 后台守护 (execution-control)。全程监听，卡死自动恢复或降级。
8. 记忆更新 (memory)。回答发给你之后，后台悄悄记偏好，不堵主流程。

### 7 种回答模式

同一个问题，对不同的人、不同的目标，回答的形态完全不同。strategy 模块根据你的身份和意图自动选。

| 模式 | 什么情况下用 | 回答长什么样 |
|------|-----------|-------------|
| FAST | 查个简单事实 | 直接答案，加一句理由 |
| STANDARD | 默认 | 结论、分析、理由、下一步 |
| DEEP | 复杂设计或高风险决策 | 拆解、多方案对比、推荐、风险、验证方法 |
| COACH | 想学东西、理解概念 | 直觉引入、推导、例子、常见误区、可迁移方法 |
| EXECUTIVE | 给领导看 | 结论开篇、影响、优先级、行动项 |
| CREATIVE | 起名字、做设计、头脑风暴 | 先发散至少 8 个方向，再收敛到 TOP3 |
| DEBUG | 修 bug | 症状、假设（按可验证性排列）、根因、修复、验证 |

- 格式校验：235 条用例全过，秒级自动跑。
- 语义测试：15 条真实 agent 实测全过，覆盖意图解析、风险判断、模式路由、质量检查、故障恢复。
- 辩论系统：3 个真实案例端到端验证。不同视角会产生不同判断，分歧能被抽出来，弱的方案会被拒绝。

## 和其他 skill 的关系

collab-master 和 [superpowers](https://github.com/obra/superpowers) 这类 skill 出发点一样：把 AI 的固定失败方式命名出来，然后写一个让它无处遁形的程序。但两者工作在不同的层。

**superpowers 是横向纪律库**——每个 skill 管一种具体情境，在那个动作内部执法：
- "修 bug 之前，先找根因，不许直接改。"
- "说测试通过之前，把命令输出贴出来。"

每个 skill 是原子的、独立的，以工程工作流为主。

**collab-master 是纵向调度器**——管的是整个任务全程的形状：
- 该追问还是按默认假设往下走？
- 该给多个方案还是直接给结论？
- 这个任务风险高到需要联网核实吗？
- 产出该发给下游 PPT 工具还是留成文字？

它不告诉 AI"怎么调试"。它决定的是这个任务需要调试这件事本身、调试之前该收集什么信息、答案该长什么样、以及结果在到达你之前有没有通过质量检查。

两个层叠在一起刚好互补：

```
collab-master 决定 →  "这是个调试任务，整体这么走"
                              ↓
superpowers 填入 →    "调试这一步，用这套纪律"
```

**它们不是竞争关系**。superpowers 管动作内部的"怎么做对"；collab-master 管动作外部的"做什么、什么时候做、做完干嘛"。装了两个的话，superpowers 在每个具体动作里执法，collab-master 在这些动作外面设路由和检查闸。


**覆盖范围的实际差别**：superpowers 聚焦在工程工作流（调试、代码审查、git、TDD）。collab-master 不分领域——同一套调度逻辑处理分析、写作、决策、创意、研究、工程。

| | superpowers | collab-master |
|---|---|---|
| 工作层 | 动作内部纪律 | 跨动作编排 |
| 覆盖范围 | 工程工作流 | 任意任务类型 |
| 粒度 | 每种情境一个原子 skill | 全会话控制器 |
| 对抗的失败 | 动作内部的错误程序 | 整个任务形状走偏 |
| 关系 | 应用/库 | 操作系统/调度器 |

## 常见问题

**装了之后所有问题都会变慢吗？**

不会。简单问题（1+1、今天几号、什么是递归）第一层就判定为低复杂度，大部分模块直接跳过，几乎不增加延迟。

**它和 prompt 优化器有什么区别？**

prompt 优化器改写你的问题来让 AI 答得更好。collab-master 不改你的问题。它改的是流程：要不要追问、要不要标假设、要不要发散、要不要转给另一个工具。它控制的是 AI 的行为，不改你的措辞。

**它自己会做 PPT、画图、写代码吗？**

不会。它自己不产出最终内容。但如果你要做 PPT，它会找你能用的下游工具，把分析材料转好格式发过去。它只负责调度，不负责生成。

**我怎么感觉到它起作用了？**

两种信号。信息不够的时候它会追一个问题，不乱编。答案里会出现 [假设:...] 和 [待验证:...] 标记，它清楚自己哪里不确定，不假装。
